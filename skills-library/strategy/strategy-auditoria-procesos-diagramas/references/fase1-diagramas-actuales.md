# Fase 1 — Formato de presentación

Para cada proceso identificado en la transcripción:

```
## PROCESO [N]: [NOMBRE DEL PROCESO]

**Objetivo:** [Descripción breve del objetivo]

**Situación actual identificada:**
- [Punto clave 1]
- [Punto clave 2]
- [etc.]

**Diagrama del flujo actual:**
[Código Mermaid.js del diagrama — PALETA A — con leyenda integrada]

**Observaciones iniciales:**
- ⚠️ Fricciones detectadas: [lista]
- 🤖 Oportunidades de automatización evidentes: [lista]
- 👤 Dependencias manuales críticas: [lista]
```

## Niveles de madurez digital

- 🔴 **Básico**: procesos mayormente manuales, pocas herramientas digitales, equipo sin experiencia en automatización
- 🟡 **Intermedio**: mix de manual y digital, algunas automatizaciones simples, equipo con disposición al cambio
- 🟢 **Avanzado**: stack tecnológico sólido, automatizaciones existentes, equipo tech-friendly

## Bloque de cierre — Diagnóstico Inicial

Al final de la Fase 1, antes de pasar a las preguntas de la Fase 2:

```
---
## DIAGNÓSTICO INICIAL

**Procesos identificados:** [X]
**Nivel de madurez digital estimado:** [🔴 Básico / 🟡 Intermedio / 🟢 Avanzado]
**Justificación:** [2-3 frases explicando el nivel asignado]

**Top 3 fricciones críticas detectadas:**
1. [Fricción más grave]
2. [Segunda fricción]
3. [Tercera fricción]

**Potencial de automatización estimado:** [Alto / Medio / Bajo] — [una frase de justificación]
---
```

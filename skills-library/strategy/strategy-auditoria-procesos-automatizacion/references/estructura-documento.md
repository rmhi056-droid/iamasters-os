# Estructura del documento final (12 secciones)

**Encabezado:**
```
BATERÍA DE PREGUNTAS PARA AUDITORÍA DE AUTOMATIZACIÓN
Empresa: [Nombre]
Sector: [Sector]
Fecha de preparación: [Fecha actual]
Preparado para: Consultoría de automatización con IA
```

**1. Resumen ejecutivo** — contexto de la empresa (2-3 párrafos), objetivos principales de la auditoría, áreas de foco identificadas

**2. Estrategia de la consultoría** — enfoque recomendado para la reunión, orden sugerido de temas, tiempo estimado por sección, consejos para generar rapport y confianza

**3. Preguntas estratégicas generales** (8-12 preguntas) — visión y objetivos empresariales, desafíos actuales principales, prioridades a corto/medio plazo, expectativas respecto a la automatización

**4. Análisis por departamentos/áreas** — mínimo 3, máximo 6 áreas relevantes según el contexto. Para cada área:

- **[Nombre del área]** (Ej: Ventas y Comercial, Operaciones, Atención al Cliente...)
- *Objetivo del análisis*: qué se busca descubrir en esa área
- *Procesos típicos a explorar*: 4-6 procesos comunes del área
- *Preguntas específicas* (15-25 por área), siguiendo este patrón de exploración: mapeo del proceso actual → tareas manuales → puntos de dolor → volumen y frecuencia → herramientas actuales → flujos de información → dependencias y aprobaciones → errores y excepciones → oportunidad de mejora

**5. Preguntas técnicas y de infraestructura** (10-15) — stack tecnológico detallado, integraciones existentes, capacidad técnica del equipo, restricciones de seguridad y compliance, acceso a datos y APIs

**6. Preguntas sobre recursos y viabilidad** (8-10) — presupuesto y recursos disponibles, capacidad de inversión en tiempo del equipo, sponsors y stakeholders clave, procesos de decisión y aprobación, experiencias previas con proyectos similares

**7. Preguntas de cierre y próximos pasos** (5-7) — expectativas post-auditoría, timeline de decisión, criterios de éxito, competidores o alternativas evaluadas

**8. Checklist de información a solicitar** — documentos, accesos o info a pedir (diagramas de procesos, ejemplos de reportes/documentos manuales, acceso demo a herramientas, métricas actuales de tiempos/volúmenes/costes, otros según contexto)

**9. Señales de alerta y oportunidades clave**
- Señales de ALTA oportunidad de automatización (5-7)
- Señales de ALERTA / posibles obstáculos (5-7)

**10. Framework de priorización** — para cada proceso identificado: Impacto (Alto/Medio/Bajo), Complejidad técnica (Baja/Media/Alta), Quick win (¿implementable en <4 semanas?), ROI estimado (¿retorno <6 meses?). Priorizar Alto Impacto + Baja Complejidad = quick wins a proponer primero

**11. Tips y mejores prácticas para la reunión** (5-7) — técnicas de entrevista, cómo identificar info relevante que el cliente no sabe que lo es, cómo manejar objeciones/resistencia al cambio

**12. Plantilla de notas** — tabla/lista para tomar notas en la reunión: proceso identificado, frecuencia, tiempo actual, personas involucradas, complejidad percibida, prioridad, notas adicionales

## Formato de salida

Markdown claro, fácil de copiar/imprimir/convertir a PDF: encabezados (##, ###), listas numeradas para preguntas, checkboxes `[ ]` para checklists, negritas para conceptos clave, cursivas para notas/tips, bloques de código para plantillas.
